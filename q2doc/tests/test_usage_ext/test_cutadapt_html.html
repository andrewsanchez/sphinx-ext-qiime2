Download and import data used in this tutorial¶

Note

the forward.fastq.gz and metadata.tsv files will be stored under the data/ dir, which will be the extent of the data scope. Everything stored in data/ will get download blocks generated automatically.

First, we declare data needed by the usage example that follows. This will render download elements. At this point, we don’t mention anything driver-specific - just get the data.

Demultiplex the reads¶

Now that we have everything imported, let’s do the thing:

Choose your interface:
  • CLI
  • Artifact API

Command Line Interface 

$ qiime cutadapt demux-single \
    --i-seqs multiplexed_sequences.qza \
    --p-error-rate 0 \
    --m-barcodes-file metadata.tsv \
    --m-barcodes-column 'barcodes' \
    --o-per-sample-sequences demultiplexed_seqs.qza \
    --o-untrimmed-sequences untrimmed.qza

Artifact API 

from qiime2 import Artifact
from qiime2 import Metadata

multiplexed_sequences = Artifact.load('multiplexed_sequences.qza')
metadata = Metadata.load('metadata.qza')

from qiime2.plugins.cutadapt.methods import demux_single

barcodes = metadata.get_column('barcodes')

demultiplexed_seqs, untrimmed = demux_single(
    seqs=multiplexed_sequences,
    barcodes=barcodes,
    error_rate=0,
)

As you can see - the reads are now demuxed. Enjoy!
